# CryptoSafe Manager
Безопасный менеджер паролей с открытым исходным кодом и шифрованием AES-256-GCM.


## О проекте
CryptoSafe Manager — это десктопное приложение для хранения паролей с шифрованием, аудитом действий и безопасным буфером обмена. Проект разрабатывается модульно, чтобы каждый компонент можно было улучшать независимо.

### Реализованные возможности

#### Sprint 1 (Фундамент)
- Хранение паролей в зашифрованной SQLite базе
- Мастер первого запуска (создание мастер-пароля)
- Базовый интерфейс с таблицей записей
- Добавление/удаление записей
- Система событий для слабосвязанной архитектуры
- Резервное копирование базы данных

#### Sprint 2 (Аутентификация)
- Мастер-пароль с хэшированием Argon2id
- Ключ шифрования через PBKDF2-HMAC-SHA256
- Сессионное кэширование ключей в памяти
- Exponential backoff при неудачных попытках входа
- Смена мастер-пароля с ре-шифрованием всех записей

#### Sprint 3 (Vault CRUD + Шифрование)
- Популярное шифрование AES-256-GCM для каждой записи с аутентификацией
- Полные CRUD операции с поддержкой транзакций
- Безопасный генератор паролей (8-64 символа, наборы символов)
- Индикатор надёжности пароля (интеграция с zxcvbn)
- Таблица с сортировкой, множественным выбором, контекстным меню
- Мгновенный поиск по всем полям с нечётким соответствием
- Мягкое удаление с возможностью восстановления

#### Sprint 4 (Безопасный буфер обмена) - НОВОЕ!
- **Автоочистка клипборда** через настраиваемый таймаут (5-300 секунд)
- **Индикатор статуса** с обратным отсчётом в статус-баре
- **Платформ-специфичная защита:**
  - Windows: win32clipboard API с `CryptProtectMemory`
  - macOS: NSPasteboard с приватным буфером
  - Linux: xclip/xsel с поддержкой Wayland
- **Мониторинг угроз:**
  - Детектирование внешнего доступа к клипборду
  - Автоускорение очистки при подозрительной активности
  - Блокировка операций при превышении порога (5+ подозрений)
- **Очистка при:** таймауте, ручном сбросе, блокировке Vault, новом копировании
- **Защита памяти:** обфускация данных в RAM, нулирование после использования
- **Уведомления:** тост-уведомления при копировании и скорой очистке (за 5 секунд)

### В плане (Sprint 5-8)
- Аудит действий с цифровой подписью (Sprint 5)
- Двухфакторная аутентификация (TOTP) (Bonus)
- Автоматическая блокировка при бездействии (Sprint 7)
- Безопасный шаринг записей (Sprint 6)
- Режим паники (Panic Mode) (Sprint 7)
- Экспорт/импорт в зашифрованном формате (Sprint 8)

## Быстрый старт

### Установка
```bash
# Клонируем репозиторий
git clone https://github.com/Katarsis435/cryptosafe-manager.git
cd PW

# Создаём виртуальное окружение
python -m venv venv

# Активируем
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# Ставим зависимости
pip install -r requirements.txt

# Для Sprint 4 (рекомендуется):
# Windows:
pip install pywin32
# macOS:
pip install pyobjc
# Linux (опционально):
sudo apt install xclip  # или xsel

# Запуск
python main.py
```

При первом запуске откроется мастер настройки:
- Придумайте мастер-пароль (минимум 12 символов, с цифрами и спецсимволами)
- Выберите где хранить базу данных

### Тестирование
```bash
# Запустить все тесты
python -m unittest discover tests

# С проверкой покрытия
pip install coverage
coverage run -m unittest discover tests
coverage report
```

## Технологии
- **Python 3.8+** (разработано на 3.13)
- **SQLite** (встроенная БД с WAL-режимом)
- **tkinter** (интерфейс)
- **cryptography** (AES-256-GCM, PBKDF2)
- **argon2-cffi** (хэширование мастер-пароля)
- **pyperclip** (кросс-платформенный буфер обмена)
- **pywin32** / **pyobjc** (платформ-специфичная защита)
- **zxcvbn** (оценка стойкости паролей, опционально)
- **unittest** (тесты)

## Безопасность
- **AES-256-GCM** с уникальным nonce для каждой записи
- **Argon2id** для хэширования мастер-пароля
- **PBKDF2-HMAC-SHA256** для ключа шифрования (100,000 итераций)
- **Экспоненциальная задержка** при неудачных попытках входа
- **Обфускация данных в RAM**, нулирование после использования
- **Автоочистка буфера обмена** (5-300 секунд)
- **Детектирование внешнего доступа** к клипборду
- **Ключ шифрования никогда не сохраняется на диск**
